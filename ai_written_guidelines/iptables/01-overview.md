# Overview

## What Is Iptables?

Iptables is the user-space command-line tool for configuring the **Linux kernel packet filtering framework** (Netfilter). It controls:

- **filtering** — allow or block packets
- **NAT** — network address translation (masquerading, port forwarding)
- **packet mangling** — modifying packet headers
- **connection tracking** — stateful filtering based on connection state

Iptables is the classic tool. Its modern replacement is **nftables** (see [07-nftables.md](./07-nftables.md)), but iptables syntax is still widely used and understood.

## Architecture

```
Packet arrives
      │
      ▼
┌─────────────────────────────┐
│         Tables              │
│  ┌───────┐ ┌────┐ ┌──────┐ │
│  │ filter│ │ nat│ │mangle│ │  ← Each table has chains
│  └───┬───┘ └──┬─┘ └──┬───┘ │
│      │        │      │      │
│  ┌───▼───┐ ┌──▼──┐ ┌─▼────┐│
│  │ INPUT │ │PREROUT││INPUT ││  ← Chains contain rules
│  │ FORWARD││POSTROUT││FORWARD││
│  │ OUTPUT │ │OUTPUT││...   ││
│  └───────┘ └─────┘ └──────┘│
└─────────────────────────────┘
```

## Tables

Iptables organizes rules into **tables**. Each table serves a different purpose:

| Table | Purpose |
|-------|---------|
| `filter` | Packet filtering (allow/block) — the default table |
| `nat` | Network Address Translation (source/destination NAT) |
| `mangle` | Packet header modification (TTL, TOS, etc.) |
| `raw` | Connection tracking exemptions |
| `security` | SELinux label assignment (rarely used directly) |

## Chains

Each table contains **chains** — ordered lists of rules. Built-in chains correspond to netfilter hooks in the kernel packet path:

| Chain | Hook point | Applies to |
|-------|-----------|------------|
| `INPUT` | Local input | Packets destined for this host |
| `FORWARD` | Forward | Packets routed through this host |
| `OUTPUT` | Local output | Packets generated by this host |
| `PREROUTING` | Before routing | Packets just arrived (DNAT, raw) |
| `POSTROUTING` | After routing | Packets about to leave (SNAT, masquerade) |

## Rules

A rule is a set of **matches** and a **target**:

```
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/24 -j ACCEPT
         │      │      │           │             │
         │      │      │           │             └── target (what to do)
         │      │      │           └── match (destination port 22)
         │      │      └── match (protocol TCP)
         │      └── chain (INPUT)
         └── action (append rule)
```

A packet is checked against rules in order. The **first match wins**. If no rule matches, the chain's **policy** (default action) applies.

## Targets

| Target | Effect |
|--------|--------|
| `ACCEPT` | Allow the packet |
| `DROP` | Silently discard the packet (no reply) |
| `REJECT` | Discard and reply with an error (icmp-port-unreachable) |
| `LOG` | Log the packet and continue to next rule |
| `RETURN` | Stop processing this chain, return to calling chain |
| `DNAT` | Destination NAT (PREROUTING chain) |
| `SNAT` | Source NAT (POSTROUTING chain) |
| `MASQUERADE` | Dynamic source NAT (for dial-up / DHCP interfaces) |
| `REDIRECT` | Redirect to local port (transparent proxy) |

## Stateful vs Stateless

A **stateless** firewall checks each packet in isolation. A **stateful** firewall tracks connections and makes decisions based on connection state (NEW, ESTABLISHED, RELATED, INVALID). Iptables uses the `conntrack` module for stateful filtering (see [04-connection-tracking.md](./04-connection-tracking.md)).

## Saving Rules

Iptables rules are runtime-only by default. They need to be saved to survive reboot:

```bash
# Debian/Ubuntu
sudo apt install iptables-persistent
sudo iptables-save > /etc/iptables/rules.v4

# Fedora/RHEL
sudo dnf install iptables-services
sudo service iptables save
```

Or use `iptables-persistent` / `netfilter-persistent`.
