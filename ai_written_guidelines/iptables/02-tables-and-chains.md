# Tables and Chains

## The Five Tables

### filter (default)

The main table for allowing or blocking packets. Used most of the time.

Built-in chains:
- **INPUT** — packets destined for this host
- **FORWARD** — packets routed through this host (gateway/router)
- **OUTPUT** — packets generated by this host

```bash
# List filter rules (default table, -t filter is optional)
sudo iptables -L -n -v
```

### nat

Network Address Translation — modifying source or destination addresses.

Built-in chains:
- **PREROUTING** — alter destination address before routing (DNAT)
- **POSTROUTING** — alter source address before leaving (SNAT, MASQUERADE)
- **OUTPUT** — alter destination for locally generated packets

```bash
sudo iptables -t nat -L -n -v
```

### mangle

Packet header modification. Rarely needed for basic setups.

Built-in chains:
- **PREROUTING** — before routing
- **INPUT** — before local delivery
- **FORWARD** — forwarded packets
- **OUTPUT** — locally generated
- **POSTROUTING** — before leaving

```bash
sudo iptables -t mangle -L -n -v
```

### raw

Exempt packets from connection tracking. Use before PREROUTING and OUTPUT to improve performance for high-traffic ports that don't need stateful filtering.

Built-in chains:
- **PREROUTING** — before connection tracking
- **OUTPUT** — before connection tracking for local packets

```bash
sudo iptables -t raw -L -n -v
```

### security

Used for SELinux label assignment. Not commonly used directly.

## Default Policies

Each chain has a **default policy** that applies when no rule matches:

```bash
# Set default policy
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# View policies
sudo iptables -L -n -v
# Look for the "(policy DROP)" or "(policy ACCEPT)" line
```

## Custom Chains

Create custom chains for organization:

```bash
# Create a chain
sudo iptables -N SSH_RULES

# Add rules to it
sudo iptables -A SSH_RULES -s 10.0.0.0/24 -j ACCEPT
sudo iptables -A SSH_RULES -s 192.168.1.0/24 -j ACCEPT
sudo iptables -A SSH_RULES -j LOG --log-prefix "SSH_BLOCKED "
sudo iptables -A SSH_RULES -j DROP

# Jump to custom chain from built-in chain
sudo iptables -A INPUT -p tcp --dport 22 -j SSH_RULES
```

## Rule Order Matters

Rules are evaluated top-to-bottom. The **first match wins**. Insert rules at a specific position:

```bash
# Insert at position 1 (top)
sudo iptables -I INPUT 1 -s 10.0.0.5 -j ACCEPT

# Append to end
sudo iptables -A INPUT -s 10.0.0.6 -j ACCEPT

# Replace rule at position 3
sudo iptables -R INPUT 3 -s 10.0.0.7 -j ACCEPT

# Delete rule at position 5
sudo iptables -D INPUT 5
```

A common pattern:

```
1. Allow established connections (stateful)
2. Allow specific trusted IPs
3. Allow specific services
4. Log and drop everything else
```
