# Packet Flow

Understanding how packets traverse iptables tables and chains is essential for writing correct rules.

## Overview

```
                    ┌──────────────────────┐
                    │      Network         │
                    └─────────┬────────────┘
                              │
                    ┌─────────▼────────────┐
         ┌────────►│      PREROUTING      │◄─────────┐
         │         │   raw, mangle, nat   │          │
         │         └─────────┬────────────┘          │
         │                   │                        │
         │          ┌────────▼───────┐               │
         │          │    Routing     │               │
         │          │   Decision     │               │
         │          └──┬─────────┬──┘               │
         │             │         │                    │
         │    local    │         │   not for us       │
         │   delivery  │         │  (forwarding)      │
         │             ▼         ▼                    │
         │    ┌────────────┐   ┌────────────────┐    │
         │    │   INPUT    │   │   FORWARD      │    │
         │    │ mangle,    │   │ mangle, filter │    │
         │    │ filter     │   │                │    │
         │    └─────┬──────┘   └───────┬────────┘    │
         │          │                  │              │
         │          ▼                  ▼              │
         │    ┌────────────┐   ┌────────────────┐    │
         │    │ Local      │   │   POSTROUTING  │    │
         │    │ Process    │   │ mangle, nat    │    │
         │    └──────┬─────┘   └───────┬────────┘    │
         │           │                 │              │
         │    ┌──────▼──────┐  ┌──────▼───────┐      │
         └────┤   OUTPUT    │  │    Network   │      │
              │ raw, mangle,│  └──────────────┘      │
              │ nat, filter │                         │
              └──────┬──────┘                         │
                     │                                │
                     └────────────────────────────────┘
```

## Path for Different Packet Types

### Packet destined for this host (INPUT)

```
PREROUTING (raw, mangle, nat) → Routing Decision → INPUT (mangle, filter) → Local Process
```

### Packet generated by this host (OUTPUT)

```
Local Process → OUTPUT (raw, mangle, nat, filter) → Routing Decision → POSTROUTING (mangle, nat) → Network
```

### Packet forwarded through this host (FORWARD)

```
PREROUTING (raw, mangle, nat) → Routing Decision → FORWARD (mangle, filter) → POSTROUTING (mangle, nat) → Network
```

## What Happens at Each Point

### PREROUTING

- **raw** — `NOTRACK` target to exempt from connection tracking
- **mangle** — packet header changes before routing
- **nat** — `DNAT` to change destination address (port forwarding)

The packet hasn't been routed yet — the destination IP is still the original one.

### Routing Decision

The kernel looks up the destination IP in its routing table to decide:
- Send to local INPUT chain (destination is this host)
- Send to FORWARD chain (destination is another host)
- Send to OUTPUT chain (packet was generated locally)

### INPUT

- **mangle** — header changes for local delivery
- **filter** — allow or block local delivery

### FORWARD

- **mangle** — header changes for forwarded packets
- **filter** — allow or block forwarding

### OUTPUT

- **raw** — `NOTRACK` for locally generated packets
- **mangle** — header changes for outgoing packets
- **nat** — `DNAT` to change destination of locally generated packets
- **filter** — allow or block outgoing packets

### POSTROUTING

- **mangle** — header changes before leaving
- **nat** — `SNAT` or `MASQUERADE` to change source address

### Routing Decision (again for OUTPUT)

Locally generated packets get a second routing decision after OUTPUT, before POSTROUTING. This allows NAT in OUTPUT to affect routing.

## Practical Implications

### Where to put DNAT rules

DNAT goes in **PREROUTING** (not INPUT) because the destination must be changed before the routing decision.

### Where to put SNAT/Masquerade rules

SNAT goes in **POSTROUTING** (not FORWARD) because the source address is set just before the packet leaves.

### Where to block forwarded traffic

Block in **FORWARD** (filter table), not INPUT or OUTPUT. INPUT is only for traffic to this host.

### Where connection tracking matters

Connection tracking happens **before PREROUTING** (before any table). The `raw` table's `NOTRACK` target is the only way to skip it.

### Common Mistake: Blocking in INPUT for forwarded traffic

A router/gateway must use FORWARD rules, not INPUT rules:

```bash
# WRONG — this blocks forwarded traffic from reaching INPUT
# (forwarded traffic never goes through INPUT)
sudo iptables -A INPUT -s 10.0.0.0/24 -j DROP

# CORRECT — block forwarded traffic in FORWARD chain
sudo iptables -A FORWARD -s 10.0.0.0/24 -j DROP
```
