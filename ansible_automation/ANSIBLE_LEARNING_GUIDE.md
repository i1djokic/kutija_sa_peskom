## Getting Started

### Prerequisites
- Complete the setup: `./setup-linux.sh`
- Log out and back in (or `newgrp libvirt`)
- Start VMs: `vagrant up --provider=libvirt`
- For multiple VMs: `VM_COUNT=2 vagrant up --provider=libvirt`
- Generate SSH config: `./scripts/generate-ssh-config.sh`
- To use dynamic inventory: `ansible -i scripts/vagrant-inventory.sh -m ping`

### First Ansible Command

Test connection to all VMs:
```bash
ansible all -m ping
```

Expected output:
```
ansible-target-1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
```

### Connect to a VM via SSH
```bash
vagrant ssh ansible-target-1
```

## Core Concepts

### What is Ansible?
Ansible is an automation engine that automates cloud provisioning, configuration management, and application deployment.

### Key Components

1. **Control Node**: Machine where Ansible is installed (your Linux host)
2. **Managed Nodes**: Target machines managed by Ansible (your VMs)
3. **Inventory**: List of managed nodes (`inventory/vagrant/hosts.yml`)
4. **Playbooks**: YAML files containing automation tasks
5. **Modules**: Units of code that Ansible executes (ping, apt, copy, etc.)
6. **Tasks**: Actions executed by Ansible using modules
7. **Plays**: Mapping of hosts to tasks
8. **Roles**: Reusable collections of tasks, handlers, templates, and vars

### Architecture
```
Linux Host (Control Node)
    |
    ├── ansible / ansible-playbook
    ├── inventory/vagrant/hosts.yml
    ├── .vagrant/ssh-config (generated by scripts/generate-ssh-config.sh)
    └── SSH ──> VM1 (ansible-target-1)
              └── VM2 (ansible-target-2) [if VM_COUNT=2]
```

## Basic Commands

### Ad-Hoc Commands

```bash
# Ping all hosts
ansible all -m ping

# Run command on all hosts
ansible all -m shell -a "uptime"

# Check disk space
ansible all -m shell -a "df -h"

# Install package on all hosts
ansible all -m apt -a "name=nginx state=present" --become

# Gather facts
ansible all -m setup

# Filter facts
ansible all -m setup -a "filter=ansible_distribution*"
```

### Running Playbooks

```bash
# Run Vagrant playbook (packages + apache)
ansible-playbook playbooks/vagrant.yml

# Run with tags
ansible-playbook playbooks/vagrant.yml --tags "apache"

# Run on specific host
ansible-playbook playbooks/vagrant.yml --limit ansible-target-1

# Check mode (dry run)
ansible-playbook playbooks/vagrant.yml --check

# Diff mode (show file changes)
ansible-playbook playbooks/vagrant.yml --diff

# Verbose output
ansible-playbook playbooks/vagrant.yml -vvv
```

### Using Different Inventories

```bash
# Production inventory
ansible all -i inventory/production/hosts.yml -m ping
ansible-playbook -i inventory/production/hosts.yml playbooks/site.yml

# Staging inventory
ansible all -i inventory/staging/hosts.yml -m ping
ansible-playbook -i inventory/staging/hosts.yml playbooks/site.yml

# Dynamic inventory (auto-detects running VMs)
ansible -i scripts/vagrant-inventory.sh -m ping
ansible-playbook -i scripts/vagrant-inventory.sh playbooks/vagrant.yml
```

## Practice Exercises

### Beginner Level

**Exercise 1: First Ping**
```bash
ansible all -m ping
```

**Exercise 2: Gather System Info**
```bash
ansible all -m setup -a "filter=ansible_hostname"
```

**Exercise 3: Run Ad-Hoc Command**
```bash
ansible all -m shell -a "uname -a"
```

**Exercise 4: Create a Simple Playbook**
Create `playbooks/hello.yml`:
```yaml
---
- name: Hello World Playbook
  hosts: all
  tasks:
    - name: Print message
      debug:
        msg: "Hello from Ansible!"
```

Run it:
```bash
ansible-playbook playbooks/hello.yml
```

**Exercise 5: Run the roles**
```bash
# Install only packages
ansible-playbook playbooks/packages.yml

# Configure only Apache
ansible-playbook playbooks/apache.yml

# Full setup
ansible-playbook playbooks/vagrant.yml
```

### Intermediate Level

**Exercise 6: Customize Apache port**
Edit `inventory/vagrant/group_vars/all.yml`:
```yaml
apache_listen_port: 8080
```
Then run: `ansible-playbook playbooks/vagrant.yml --tags apache`

**Exercise 7: Add packages**
Edit `inventory/vagrant/group_vars/all.yml` to add `nginx` to `base_packages`.

**Exercise 8: Use Conditions**
Create a playbook that installs different packages on different hosts.

**Exercise 9: Create a new role**
Structure:
```
roles/ntp/
├── tasks/main.yml
├── handlers/main.yml
└── defaults/main.yml
```

### Advanced Level

**Exercise 10: Templates**
Modify `roles/apache/templates/index.html.j2` to show custom content.

**Exercise 11: Multi-VM Orchestration**
Start 3 VMs (`VM_COUNT=3 vagrant up`) and configure them differently using host vars.

**Exercise 12: Error Handling**
Use `ignore_errors`, `failed_when`, and `register` to handle task failures.

## Quick Reference

### Common Modules

| Module | Description | Example |
|--------|-------------|---------|
| ping | Test connectivity | `ansible all -m ping` |
| shell | Run shell command | `-m shell -a "ls -la"` |
| apt | Manage packages (Debian) | `-m apt -a "name=nginx state=present"` |
| copy | Copy files | `-m copy -a "src=file dest=/tmp/"` |
| template | Deploy templates | `-m template -a "src=template.j2 dest=/etc/config"` |
| service | Manage services | `-m service -a "name=nginx state=started"` |
| file | Manage files/dirs | `-m file -a "path=/tmp/test state=touch"` |
| debug | Print debug info | `-m debug -a "var=ansible_hostname"` |

### Playbook Structure
```yaml
---
- name: Playbook name
  hosts: group_name
  become: yes
  vars:
    my_var: value
  tasks:
    - name: Task description
      module_name:
        option: value
      when: condition
  handlers:
    - name: Handler description
      module_name:
        option: value
```

### Role Structure
```
roles/rolename/
├── defaults/main.yml       # Default variables (lowest precedence)
├── vars/main.yml           # Internal variables
├── tasks/main.yml          # Main task list
├── handlers/main.yml       # Handlers (notified by tasks)
├── templates/              # Jinja2 templates
├── files/                  # Static files to copy
├── meta/main.yml           # Role metadata & dependencies
└── README.md               # Role documentation
```
